# Daily Feed Newsletter

Daily Feed sends one polished email every morning with:

- a short inspirational quote
- a guitar song suggestion with chords and difficulty
- an outdoor adventure recommendation in Lebanon

Content is generated by OpenAI using a structured prompt, then formatted into a clean plain-text + HTML newsletter.

## How It Works

Each run:

1. Calls OpenAI to generate a JSON payload with `quote`, `song`, and `adventure`
2. Uses prompt guidance to keep content fresh and different from prior days
3. Renders a minimal product-style newsletter layout
4. Sends email through Gmail SMTP

No local persistence file is required.

## Project Structure

- `main.py` - orchestrates content generation, formatting, and sending
- `services/openai_service.py` - OpenAI API integration and JSON parsing
- `services/email_service.py` - Gmail SMTP sender (plain + HTML email)
- `.github/workflows/daily.yml` - automation workflow

## Technologies Used

- Python 3.11
- Python standard library:
  - `urllib.request` (OpenAI HTTP request)
  - `json`
  - `smtplib`
  - `email.message`
  - `datetime`
- OpenAI Chat Completions API
- GitHub Actions for scheduling
- Gmail SMTP (`smtp.gmail.com:587`)

## Local Run

Create `services/.env` (local only, never commit secrets):

```env
DAILY_FEED_EMAIL="you@gmail.com"
DAILY_FEED_PASSWORD="your_gmail_app_password"
DAILY_FEED_RECIPIENT="you@gmail.com"
OPENAI_API_KEY="your_openai_api_key"
```

Then run:

```bash
python main.py
```

## GitHub Actions Automation

Workflow file: `.github/workflows/daily.yml`

It runs daily using cron and can also be triggered manually.

Required repository secrets:

- `EMAIL_USER` - sender Gmail address
- `EMAIL_PASS` - Gmail App Password
- `OPENAI_API_KEY` - OpenAI API key

Mapped runtime variables:

- `DAILY_FEED_EMAIL`
- `DAILY_FEED_PASSWORD`
- `DAILY_FEED_RECIPIENT`
- `OPENAI_API_KEY`

## Is It Fully Automatic?

Yes. Once the workflow is enabled and secrets are set, GitHub Actions runs daily (UTC schedule) and sends the email automatically.
